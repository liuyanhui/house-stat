# 北京房地产数据抓取程序

从北京市住建委网站自动抓取房地产签约数据。

[![Python](https://img.shields.io/badge/Python-3.x-blue.svg)](https://www.python.org)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

## 功能

**二手房数据**：经纪机构排行、区县/面积/价格段统计、每日/月度数据、五年历史

**新房数据**：每日签约统计、五年网签数据

**特性**：智能去重、自动重试、日志记录、向后兼容

## 数据来源

北京市住建委：http://bjjs.zjw.beijing.gov.cn/eportal/ui?pageId=307749

## 安装

```bash
pip install -r requirements.txt
```

## 使用

```bash
python main.py
```

程序自动抓取数据并保存到 `data/` 目录，已存在的数据会自动跳过。抓取完成后自动运行数据完整性校验（面积/价格段加总 vs 全市），不一致则非零退出。

**趋势分析报告**（Markdown + HTML + PNG 图表，写入 `report/`）：

```bash
python script/analyze.py --report
```

产物：`report/trend_report.md`（文本）、`report/trend_report.html`（自包含，图片 base64 内嵌，浏览器直接打开）、`report/*.png`（10 张图）。也可单独把 md 转 html：`python script/gen_html.py`。`report/` 为代码生成的衍生物，**已加入 .gitignore**，每次运行重新生成。

**控制台文本分析**（原行为，保留）：

```bash
python script/analyze.py
```

**独立数据校验**：

```bash
python script/validate.py
```

> ⚠️ **Python 版本**：本项目依赖在 **Python 3.13** 下完整可用（`requests/bs4/pandas/chinese_calendar/matplotlib`）。请用 `py -3.13` 运行。`run.bat` / `report.bat` 已默认使用 `py -3.13`。若环境用默认 `python`（3.14）且通过 `PYTHONPATH` 共享 3.13 的包，`matplotlib` 的编译扩展会因版本不匹配而导入失败——故统一用 3.13。

| 文件 | 说明 |
|------|------|
| `analyze.py` | `--report`：生成趋势报告（Markdown + HTML + PNG，全市/周度/各区/结构趋势）。无参数：原控制台文本分析（月度概览、周度、工作日/假期、区县/面积分布、同比环比） |

## 数据来源与可靠性

- **基准数据（可靠）**：全部来自北京市住建委（pageId=307749）官方页面自爬，逐月累积。趋势报告主序列仅基于此数据，经完整性校验（面积段/价格段加总 = 全市）保证一致。
- **历史数据**：官方页面仅展示当月，无历史月度回溯接口（已查实：遗留历史查询弹窗 `/publicbjjs/*.asp` 全部失效）。故月度序列从 2025-01 起、区县从 2025-12 起。
- **第三方历史**：媒体/研究院引用的"二手住宅网签"与官方口径一致（已验证：媒体 2025 上半年 88575 套 = 本项目自爬加总）。但其历史月度数据多锁在图片或散落报道、或被微信提取码门挡，可靠性与可获取性均不达标，**未纳入主序列**。仅在趋势报告附录中以"参考"形式列出可交叉验证的半年/全年锚点（2022-2025 上半年），明确标注"非自爬"。
- **不含预测**：趋势报告仅呈现历史趋势，不做未来预测。

## 注意

- CSV使用UTF-8 with BOM编码，Excel可直接打开
- 数据抓取失败时，字段值为 -1
- 可设置定时任务自动运行（Windows任务计划或Linux cron）

## 常见问题

**Q: 程序报错"未找到表格"？**
A: 网站结构可能变化，需检查网页并修改代码

**Q: 如何查看历史数据？**
A: 直接打开CSV文件查看所有记录

**Q: 会重复抓取吗？**
A: 不会，程序自动跳过已存在数据
